283.移动零

题目描述, 看这里

题解

这题是一个快慢双指针的典型题目, 主要思路就是 保证慢指针(左指针)指向非零元素的最后一位即左指针左边全都是非零元素, 快指针(右指针)向右寻找第一个非零元素, 当快指针(右指针)向右移动并找到非零元素后, 将左右指针指向的元素互换左指针+1。

初速状态是左右指针同在0下标处, 循环结束条件是右指针遍历完所有元素。

11. 盛水最多的容器

题目描述, 看这里

题解

这是一道相撞指针(反向指针)的题目, 主要思路是最左和最右的指针相向移动, 每层循环进行时, 当前容量 = 宽度(指针之差) * min(左指针和右指针分别指向的元素), 下次移动时移动指向元素较小的那个指针, 直到两个指针相撞(循环结束条件)。

15. 三数之和

题目描述, 看这里

题解

题目要求返回不重复的三元组, 这就对解题产生了很大的阻力, 我们无法直接通过暴力的方法进行解题了。

但是如何去重呢？这里的思路是将题目给出的数组进行排序, 在刚进最外层循环时, 先判断 if(i > 0 && nums[i] == nums[i - 1]), 在进第二层循环时判断 if(j > i+1 && nums[j] == nums[ j-1]) . 这样就保证了最后给出的答案不会出现重复的三元组, 如果继续暴力求解下去的话, 需要添加 if(k > j+1 && nums[k] == nums[ k-1]).为什么双指针不用加最后一个判断呢？可以看到, 在找到符合的 j 和 k之后, 就进入了第二层循环了, 且第二层循环的 nums[j] 肯定不一样, 所以不用担心 nums[k] 会重复了

因为这里讲的主要是双指针, 那么怎样利用双指针呢？我们可以发现, 在nums[i]确定的情况下, 我们需要找 nums[j] + nums[k] = -nums[i], 如果数组已经排好了序, 那么就可以利用相撞指针来寻找 nums[j] 和 nums[k]. 这样就简化了一层循环, 使得时间复杂度变成了二次方

在利用双指针减少一层循环时, 有很多小细节, k 的定义要在第二层循环外, 第二层循环 break 的时机(当 j 与 k 重合时, 此时一定无解)。在这里贴一下官方题解的说明, 比较清楚。

当第二重循环往后枚举一个元素 b′ 时，由于 b′>b，那么满足 a+b′+c′=0 的 c′ 一定有 c′<c，即 c′ 在数组中一定出现在 c 的左侧。也就是说，我们可以从小到大枚举 b，同时从大到小枚举 c，即第二重循环和第三重循环实际上是并列的关系。

实现后看着是三层循环实际上已经简化成了两层。

正确答案：

学到的Trick

  • 双指针简化循环: 排序加双指针
  • 简化循环后内部退出时机, 要好好想清楚。
